## 局限性说明

### Mock 数据的问题

当前实现使用 **mock regimetry**（因为真实的 regimetry 是私有包）：

1. **随机聚类:**
   - 聚类 ID 是随机分配的（0-7）
   - 没有基于实际市场特征
   - 不反映真实的市场机制

2. **映射问题:**
   - 所有聚类映射到同一状态（BEAR）
   - 无法区分牛市、熊市、震荡市
   - 失去了策略的自适应优势

3. **策略影响:**
   - RARS 策略假设能在不同机制间切换
   - Mock 数据下策略会始终使用 BEAR 逻辑
   - 无法验证策略的真实效果

## 下一步建议

### 选项 1: 使用真实 Regimetry

如果可以获取真实的 regimetry 包：

1. **安装 regimetry:**
   ```bash
   pip install regimetry
   ```

2. **重新生成机制标签:**
   ```bash
   python scripts/regimetry_pipeline.py
   ```

3. **验证聚类质量:**
   - 检查聚类是否反映真实的市场状态
   - 确认至少有 3 个不同的交易状态（BULL/BEAR/RANGING）

### 选项 2: 使用替代机制检测

如果无法获取 regimetry，可以考虑：

1. **基于技术指标的机制检测:**
   - 使用趋势指标（ADX, MACD）
   - 波动率指标（ATR, VIX类）
   - 市场宽度指标

2. **简化的机制分类:**
   - 牛市: 价格 > MA(200) 且 MA 斜率向上
   - 熊市: 价格 < MA(200) 且 MA 斜率向下
   - 震荡: 价格在 MA 附近波动

3. **HMM 机制检测（已有实现）:**
   - 使用 `strategies/regime/regime_detector.py`
   - 基于收益率、波动率、MA 斜率的 HMM

### 选项 3: 手动标注历史机制

1. **识别关键历史时期:**
   - 2007-2008: 牛市
   - 2008-2009: 熊市（金融危机）
   - 2009-2010: 震荡转牛
   - ... 根据历史事件标注

2. **创建机制标签文件:**
   - 手动指定每个时期的机制
   - 用于策略回测和验证
